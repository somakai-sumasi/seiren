## 欠陥スコアリングの基準

### スコア計算式
欠陥スコア = 欠陥行数 × 重み係数

### 重み係数
| 欠陥カテゴリ         | 重み係数 | 説明                         |
| -------------------- | -------- | ---------------------------- |
| カプセル化違反       | 1.5      | データ漏洩は広範囲に影響     |
| 関心の分離違反       | 1.0      | 基本的な設計違反             |
| ドメインモデル不完全 | 1.2      | ビジネスロジック散逸のリスク |
| レイヤ違反           | 1.3      | アーキテクチャ崩壊のリスク   |
| interface設計違反    | 0.8      | 影響範囲は限定的             |

### 欠陥行数のカウント方法
- 欠陥に直接関与するコード行をカウント
- コメント行・空行は除外
- 連続する欠陥行は範囲で表記（例: 13-18行目）

### 重要度判定
| 合計スコア | 重要度             |
| ---------- | ------------------ |
| 0-5        | 低（改善推奨）     |
| 6-15       | 中（要改善）       |
| 16以上     | 高（緊急対応必要） |
