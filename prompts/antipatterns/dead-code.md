## デッドコード

### 定義
デッドコードとは、実行されることのないコード、または実行されても
プログラムの動作に影響を与えないコード。

### 検出基準
1. **到達不能コード**
   - return文の後のコード
   - 常にfalseとなる条件分岐内のコード
   - throwの後のコード

2. **未使用の要素**
   - 呼び出されないprivateメソッド
   - 参照されない変数や定数
   - 使用されないimport文

3. **コメントアウトされたコード**
   - 削除せずにコメントアウトで残されたコード
   - 「後で使うかも」と残された未使用コード

4. **過剰な防御コード**
   - 絶対に発生しない条件のチェック
   - フレームワークが保証する条件の再チェック

### 欠陥の影響
- コードの可読性低下
- 保守時の混乱（動作するコードとの区別が困難）
- 不要なコンパイル/解析時間
